TOOLS 2019.103

This utility AddOn is required by all other UCE AddOns, as it contains several shared functions and UI elements. Most notably are the universal "CastBarUI", "InfoBoxUI" and a universal "PopupUI". Those UI elements also come with options that can be edited via the Inspector. Besides that, this AddOn adds a lot of new functions to the core asset, those functions are required in order for the UCE AddOns to work as expected.

INSTALLATION PROCESS

Integration Steps

  1. [Required] Apply core changes as stated in the Core Changes ReadMe first.
  2. [Recommended] Remove the "AddOn Example" CS script file in uMMORPG AddOns folder, it can cause errors with AddOns (it's not required anyway, never)
  3. [Required] Note: Even if you do not plan to use the mySQL database, go to your NetworkManager on the scene and in its inspector switch your database to mySQL, then back again to SQLite and let Unity update everything.
  4. [Required] Add all objects from "Required UI" to your Canvas via drag-n-drop.
  5. [Required] Make sure not to put them into another object.
  6. [Optional] Edit the properties of "UCE_UI_Popup" and "UCE_UI_InfoBox" in the Inspector. This is especially important for text colors of the "UCE_UI_InfoBox".
  7. [Required] Add all scripts from "Required Scripts" to each one of your player prefabs as new component (Player Prefab > Inspector > Add Component).
  8. [Optional] Edit the properties of the "UCE_Popup" on your player prefab. Add up to 255 icons and up to 255 sounds of your choice to it. Those will be used by other AddOns to customize your Popup window.
  9. [Optional] Add any game objects from "Optional Prefabs" to your scene via drag-n-drop. And edit the properties in the Inspector - you can already test the Icons and Sounds this way: Start your game and walk into a prefab you placed and edited this way.
  10. [Optional] Move the content of "Plugins" folder to your projects "Plugins" folder for consistency (not required but recommended).
  11. [Optional] You can also create speed modifying Buffs and passive Skills now, but be sure to set a max Speed to each player and monster prefab. Otherwise it won't work.
  12. [Optional] If you do not set a max speed manually, all of your entities max speed will be capped at 1.5 times their original speed.

Additional Information

When updating from an older version, be sure to delete the UCE_Tools.Buff.cs - as this file is now obsolete and not used anymore!
